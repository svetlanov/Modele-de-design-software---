# Лабораторная работа 4  
## Тема: Базовое взаимодействие с памятью через переменные

В этой лабораторной я разбирала, как переменные хранятся в памяти, чем отличаются типы-значения от ссылочных типов, и что происходит во время выполнения простых фрагментов кода.

---

## Задание 1

### Код:

```csharp
int a = 5;
int b = 6;
a = b;
b = 7;
Console.WriteLine(a);
```

### Разбор:
Создаются две переменные в стеке: a = 5 и b = 6. Инструкция a = b копирует значение b (6) в переменную a. Затем b меняется на 7, но это не влияет на a, потому что тип int — это тип-значение, и копируется именно значение. В хипе ничего не создаётся. 
- В консоль выводится: **6**

---

## Задание 2

### Код:

```csharp
int a = 5;
int b = a + 6;
a = 7;
Console.WriteLine(b);
```

### Разбор:

В стеке создаются две переменные: a = 5 и b, которой присваивается значение a + 6, то есть 11. Затем переменная a изменяется на 7, но это не влияет на b, потому что b уже содержит вычисленное значение. В хипе объекты не создаются. 
- Вывод: **11**

---

## Задание 3 

### Код:

```csharp
string a = "a";
string b = a;
a = "b";
Console.WriteLine(a);
Console.WriteLine(b);
```

### Разбор:

Строки являются ссылочными типами, поэтому переменные хранят ссылки на объекты в хипе. Сначала создаётся строка "a", и обе переменные, a и b, указывают на неё. Затем переменной a присваивается новая строка "b", создаётся второй объект, и только a начинает ссылаться на него, тогда как b остаётся указывать на прежний объект "a". Поэтому в консоли выводится сначала b, затем a.
- Вывод:
```
b
a
```

## Задание 4 

### Код:

```csharp
string a = 5;
```

### Разбор:

Код не выполняется, потому что попытка присвоить значение типа int переменной типа string вызывает ошибку компиляции. Никакие инструкции не выполняются, ячейки в стеке не создаются, значения не записываются, объекты в хипе не создаются. Переменная a не появляется в памяти, итоговых значений нет, и никакого вывода в консоль не происходит — программа не запускается.

## Задание 5

### Код:

```csharp
int a = 5;
int a = 6;
```

### Разбор:

Первая строка создаёт одну ячейку в стеке и записывает туда значение 5. Вторая строка вызывает ошибку компиляции, так как переменная a уже существует. Значения в память больше не записываются, новых ячеек не создаётся, объектов в хипе нет. Итоговое значение a — 5, но программа не запускается из-за ошибки, поэтому вывода в консоль нет.
