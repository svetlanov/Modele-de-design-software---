## 1. Компилирование и выполнимый файл

### Удаление временных файлов
Я удалила папки *bin* и *obj* вручную.

После этого попыталась выполнить программу без пересборки:

```bash
dotnet run --no-build
```

### Почему возникла ошибка?
Команда выдала ошибку, так как после удаления *bin/obj* исчезли все скомпилированные файлы (.dll/.exe).  
`--no-build` запрещает компиляцию, поэтому выполнить программу было невозможно.

---

### Восстановление артефактов компиляции

```bash
dotnet build
```

После успешной компиляции в папке:

```
bin/Debug/net8.0/
```

появился выполнимый файл.

---

### Запуск исполняемого файла

- При двойном клике окно консольной программы открылось и сразу закрылось — это нормально для консольных приложений.
- Поэтому я запустила его через терминал:

```bash
./bin/Debug/net8.0/MyProject
```


```csharp

```

---

### Вопрос: что если функцию не вызвать?

Ответ:  
Функция **никогда не выполнится сама по себе**. Код внутри функции выполняется только если она вызвана. Определение функции — это только описание того, что она делает, но не её запуск.

---

### Вопрос: важен ли порядок определения функций?

В C# порядок не важен. Можно вызывать функцию, определённую ниже в файле.  

Причина: на этапе компиляции C# заранее анализирует все объявления функций.

---

### Что такое «определение функции»?

Это блок кода, в котором **описано**, что именно делает функция, например:

```csharp
void MyFunction() {
    Console.WriteLine("Hello");
}
```

Определение ≠ выполнение.  

Выполняется только вызов:

```csharp
MyFunction();
```
